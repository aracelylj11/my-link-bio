# Update README

1. Read the existing `README.md` before making changes. Follow its format and terminology when it exists.
2. Update `README.md` to describe what changed and explain how to use each new feature or route.
3. Document every new dependency, including its purpose and installation requirements.
4. Keep the README organized with these sections: **Project Overview**, **Features**, **Setup**, and **Dependencies**. Preserve useful existing content while placing it in the appropriate section when needed.
5. Verify that commands, routes, and dependency names match the implemented project.
